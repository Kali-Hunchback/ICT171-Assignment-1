Website Proposal

Wade Roberts: #32666275

The project to be undertaken is a personal portfolio site for the sake of future employment and consolidation a series of projects which I undertake on an educational, self-improvement, and personal basis additional to projects I will start in the future.

Initially, the website is proposed to be built in a few various pages. These represent typical undertakings for portfolio sites.

The site should include

  • A landing page, which displays a “headline statement”.
  • The landing page should contain some form of navigation element which should be represented on linking pages, as necessary.
  • An about me page, in which I provide some information about what I am currently undertaking regarding projects and my interests.
  • A contact page, hopefully, this can be set up to enable an email to be forwarded on to my work associated email from within the webpage
    (most likely to be undertaken as a form and handled by PHP, more research is to be completed). Within this page there will also be links for work related socials (LinkedIn, TryHackMe, GitHub to begin with).
    Where there is information available from the other services to display inside of the page (Such as TryHackMe having a profile graphic which can be embedded using HTML and JavaScript) this shall be used instead of the conventional linking methods.
  • A blog page, which serves as a container for integrating my previous (and future) Medium posts which primarily contain TryHackMe Room write-ups.
  • The website should be responsive and available on a series of screen dimensions.

This outlines the minimum I would like to implement. There is a possible expansion of services and consideration has begun on whether other elements such as an RDBMS will be necessary to facilitate what is planned but this will be decided throughout the life cycle of the project.
